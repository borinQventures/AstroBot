## 2024-03-24 - Accessibility Labels for Search Input
**Learning:** Found an accessibility issue pattern with search inputs and icon-only close buttons lacking screen-reader accessible labels (`aria-label`) and having visible but decorative icons that should be hidden from screen readers (`aria-hidden="true"`).
**Action:** When working on JavaScript-generated UI elements, actively search for inputs and buttons, especially icon-only buttons, to ensure they have appropriate `aria-label` attributes and that inner decorative icons use `aria-hidden="true"`.
## 2024-05-24 - Search Loading & Filter Button Accessibility
**Learning:** Adding `opacity: 0` and `pointer-events: none` to an element hides it visually but leaves it in the tab order and accessible to screen readers, causing a confusing UX for keyboard and screen reader users. Also, dynamic loading states should always be explicitly announced using ARIA live regions.
**Action:** When hiding interactive elements (like the clear filters button), sync the `disabled` state or use `aria-hidden="true"` and `tabindex="-1"` along with visual hiding. For dynamic loading containers, add `role="status"` and `aria-live="polite"` to ensure screen readers announce the loading state. Add `aria-hidden="true"` to decorative icons (like spinners) inside interactive or live regions to prevent screen readers from reading raw icon markup.
