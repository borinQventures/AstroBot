## 2024-05-24 - Command Injection in openshell execution
**Vulnerability:** Command injection was possible in `status.ts` and `logs.ts` because dynamic properties like `sandboxName` from the plugin configuration were directly interpolated into string commands passed to `exec()` (e.g., `` execAsync(\`openshell sandbox status ${sandboxName} --json\`) ``). If `sandboxName` contained shell metacharacters, arbitrary commands could be executed.
**Learning:** Using `exec()` with interpolated user or configuration input creates a severe risk of command injection, especially in a tool interacting closely with local configuration and state files.
**Prevention:** Always use `execFile()` or `spawn()` instead of `exec()`. These methods execute the binary directly and pass arguments as an array, completely bypassing the shell and avoiding any injection risks. When refactoring `exec()` to `execFile()`, also update the tests to mock `execFile` and assert against the correct arguments.
